# Claim Boundaries

This page defines what the current Perfect Closure program **does** and **does not** claim.

## Positive Claims

The repository currently claims:

1. Perfect Closure is a useful structural framework for modeling transitions from free propagation to stable closure.
2. The light-to-mass thesis can be formalized as:
   - free light as non-closing null-like propagation,
   - mass as stable self-closed phase resonance.
3. Quaternionic phase geometry provides a coherent language for closed standing-wave return.
4. Hydrogen shell-locking on \(S^3\times\mathbb R_s\) is a calibrated, falsifiable bridge architecture for closure shells.

## Explicit Non-Claims

The repository does **not** claim:

1. a proof of the Riemann Hypothesis;
2. that zeta-zero matching is primary evidence for mass generation;
3. a first-principles derivation of all fundamental constants;
4. replacement of standard quantum mechanics, QED, atomic theory, or relativity;
5. a complete theory of all particles and interactions.

## Calibration Status

Hydrogen is presently treated as a **calibrated bridge**:

- \(\mathrm{Ry}\) is used as a calibration constant;
- shell architecture and spectral relationships are tested in that calibrated setting;
- success supports structural coherence, not complete constant derivation.

## Falsifiability Standard

Any proposed bridge extension should state:

1. what is fixed in advance,
2. what is calibrated,
3. what is predicted out-of-sample,
4. what empirical result would count as failure.

Without this, results should be interpreted as descriptive compatibility rather than predictive support.


## What would count as progress?

Progress in this program would look like:

1. clearer calibrated reproduction of known spectra,
2. out-of-sample predictions for spectral shifts or degeneracies,
3. extension from hydrogen to helium-like or multi-electron systems with fixed assumptions,
4. numerical simulations where closure parameters are fixed before comparison,
5. a clearer handoff from closure-action theory into reusable operator libraries such as spectral-core.
